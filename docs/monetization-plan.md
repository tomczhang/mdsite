# mdsite 商业化方案（海外托管 SaaS 线）

> 作者：@Researcher｜日期：2026-06-20｜状态：方案/研究输入（非实现）
> 受众：@chi-zhang（决策）、@MdsiteBuilder（SaaS 实现输入）、@Reviewer（审查清单）
> 范围：仅海外托管 SaaS 变现线。开源 CLI（BYO-GitHub）保持免费无限、零变更。

---

## 0. 一页结论（TL;DR）

1. **$9.9/月 合理偏稳**：卡在 Carrd（便宜）与 Super/Ghost（贵）之间。建议加年付折扣（~$79–99/年，惯例"年付送 2 个月"）。
2. **免费档限制要按市场规律设**：靠 ①服务端水印 ②限站点数 ③锁自定义域名/高级模板 转化——**不要用"每日 1 次"或"30 天删除"**（前者赶走试用者、后者毁掉建站心智，市场无人这么做）。
3. **护城河低 ≠ 赚不到钱**：Ghost 全开源仍靠 Ghost(Pro) $18/mo 跑成生意。付费人群 ≠ 会自建人群。**建议保留开源 CLI 做获客/信任，钱靠托管 SaaS 的"省心 + 专属功能"。**
4. **URL = 每用户子域名 `username.mdsite.app`**（硬性安全约束，见 §4）。免费子域名好用没关系——让人付费的不是 URL，是**去水印 + 高级能力 + 省心**。

## 1. 竞品定价实证（已验证：2026-06-20 抓官网）

| 产品 | 免费档 | 最低付费 | 免费→付费靠什么切 |
|---|---|---|---|
| **Carrd**（落地页） | 3 个站、带 "Made with Carrd" | **$19/年**（~$1.6/mo） | 去水印 + 自定义域名 + 表单 + 分析 |
| **Super.so**（Notion→站） | 1 个站、带 badge、可挑子域名 | **$16/mo**（Personal）/ $28 Pro | 去 badge + 自定义域名 + 密码页 |
| **Ghost(Pro)**（博客） | 无免费（开源可自托管） | **$18/mo**（Starter，年付价） | 托管 + 免费域名 + newsletter |

**共同规律（核心证据）**：免费档全部靠 **①品牌水印 ②限站点数 ③锁自定义域名** 转化；**没有一家**用"每日次数"或"到期删除"。

> ⚠️ 未验证：Tiiny.host / Hashnode 定价页为 JS 渲染，未抓到，未纳入。

## 2. $9.9/月 定价判断

- **位置**：> Carrd（$1.6/mo）、< Super（$16）/Ghost（$18）。对"md→专业链接、零配置"这档定位，$9.9 合理偏稳。
- **建议结构**：月付 $9.9；年付 ~$79–99（折合 $6.6–8.25/mo，市场惯例"年付送 2 个月"）。年付提现金流、降决策成本。
- **可选**：若想跑量，加 $5–6 入门档、$9.9 作 Pro。但单档 $9.9 是健康起点，不必一开始就复杂。

## 3. 免费 vs 付费 功能矩阵

**免费（获客漏斗，给非技术用户尝鲜）**
- report 模板 + 站点首页
- 带 "made with mdsite" 角标（**服务端强制**）
- `username.mdsite.app` 子域名
- **不限生成次数**、链接**永久保留（不删）**

**付费 $9.9/mo（卖"省心 + 专业感"）**
- 去水印（主力转化杠杆）
- 解锁 doc / dashboard 全部模板（= "格式分层"，主力杠杆）
- 私有 / 密码页
- 更多站点
- 自定义域名（**次要杠杆**，仅收"品牌/商业"窄人群，见 §5）
- 后续：团队协作

> **关键纠正 chi-zhang 原设想**：免费档不要"每日 1 次 + 30 天删除"。改为"不限次 + 永久 + 带水印 + 仅 report 模板"，靠水印和高级模板/功能转化。

## 4. URL / 命名空间（硬性安全约束）

**结论：每用户一个子域名 `username.mdsite.app`，不是路径 `mdsite.app/username/slug`。**

**为什么必须是子域名（安全硬约束）**
- mdsite 发布的是**用户自带的任意 HTML/JS**（doc/dashboard 跑 echarts、整页直通可塞 SPA）。
- 若所有用户共用 `mdsite.app` 一个 origin → **A 用户页面里的脚本能读/偷 B 用户的 cookie、发起同源攻击**，是真漏洞。
- 子域名 `a.mdsite.app` / `b.mdsite.app` 是**不同 origin，浏览器自动隔离**——托管用户生成内容的标准沙箱做法。
- 呼应：**开源版天然就是隔离的**（每人发到自己 `username.github.io/repo`，各自独立 origin）。SaaS 版即复刻这个隔离模型。

**为什么用户名前缀方向对**
- 与开源 GitHub Pages 心智一致（`username.github.io`），用户零理解成本。
- 市场标准（Substack / Notion / Super 免费档均为"你的名.平台.com"），给身份感 + 稳定命名空间。

**两个必须提前定的策略坑**
- **用户名抢注/冒充**：保留名单（`admin` / `api` / `www` / `help` / `app` / `mail` …）+ 注册校验，防占用知名品牌名。
- **改用户名 = 旧链接全断**：禁止改名，或改名后做 301（旧→新）重定向，避免分享出去的链接失效。

## 5. 护城河 + 反自建（chi-zhang 核心焦虑）

**事实：代码护城河 ≈ 0**（"md→自包含 HTML→Pages"一两天可复刻，@Researcher 与 @MdsiteBuilder 一致）。但不致命：

- **硬证据**：Ghost 全开源（MIT）仍靠 Ghost(Pro) $18/mo 跑成生意；WordPress 开源→WordPress.com 赚钱；Bear Blog 开源仍有付费档。
- **规律**：**付费人群 ≠ 会自建人群**。会自建的本就不付费，撤不撤开源都拦不住；付费的是"碰都不想碰 GitHub"的非技术用户，他们根本不会自建。

**结论与动作**
- **「撤回开源」不建议做**：几乎不增护城河（技术用户照样复刻），却丢掉开源两个真价值——**蹭 star 分发 + 去中心信任**。
- **真护城河 = 把"自建心理成本"做到 > $9.9/mo**：零配置、一键自定义域名、永久托管 + CDN、模板持续打磨更新、不用管 token/续费/运维。**卖的是便利和服务，不是代码。**
- **防蚕食设计**：开源 CLI 故意"留白"——只做 BYO-GitHub（用户自己有 repo、配 token、管 Pages）；SaaS 卖"这些全不用碰"。两线服务不同人群，不打架。

**关于"好子域名是否让自定义域名没人买"（chi-zhang 追问）**
- 担心成立一半：个人/爱好者用 `coolname.mdsite.app` 即够，不会为换域名付费。
- 但**不打垮模型**：Super/Notion/Substack/Carrd 都"免费给挑好子域名 + 自定义域名卖钱"，两者共存。
- 原因：**子域名是身份/handle；自定义域名是"URL 里没有平台名的自有品牌"——两种需求，子域名永远满足不了后者**。付费买域名的人要的是 URL 里彻底不出现 mdsite，子域名再好看也写着"我用了免费工具"。
- **动作**：自定义域名当**次要杠杆**（只收品牌/商业窄人群）；主力转化靠**去水印（服务端强制）+ 高级模板 + 私有页 + 省心**。让人付费的不是 URL。

## 6. 合规前置（海外托管 SaaS 线，已点过）

- 美国：Section 230 + DMCA 避风港（比大陆轻），但仍需 **ToS + 举报/删除流程 + CSAM 零容忍 + 上游 host AUP**。
- 这是 SaaS 线的前置成本，可控但真实。**开源 CLI（BYO-GitHub）不承担此责任**——内容在用户自己的 GitHub。

---

## 7. 假设与不确定性（诚实标注）

- ✅ 已验证：Carrd / Super / Ghost 定价（官网实抓）。
- ⚠️ 未验证：Tiiny.host / Hashnode 定价（JS 渲染未抓到）。
- ⚠️ **最大未验证项 = 转化率**：$9.9 的实际付费转化要靠上线后真实数据。**建议小步**：免费档 + 单一 $9.9 月付 + 年付折扣，跑 1–2 个月看转化再调。
- 架构约束（@Researcher 与 @MdsiteBuilder 独立得出一致）：限量/留存/水印这类强 gating **必须有中心化服务**，只能落在海外托管 SaaS 层，**不能加到开源 CLI**（本地代码可绕过）。
